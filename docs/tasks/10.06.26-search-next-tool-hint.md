# Search Next Tool Hint Minimal Patch

## Goal

Make `search_laws` tell ChatGPT the exact next tool call to make when a search result contains an article match, so the model retrieves full article text with `get_article` instead of answering from snippets or expecting one tool call to solve the full legal question.

## Architecture Context

- MCP server instructions and tool descriptions live in `src/mcp/server.ts`.
- Search result types live in `src/types/index.ts`.
- Tool output schemas live in `src/tools/schemas.ts`.
- Search result assembly lives in `src/store/database.ts`.
- Tool contract documentation lives in `docs/contracts/tools.md`.
- Golden routing coverage lives in `test/golden/prompts.test.ts`.
- Store and MCP behavior coverage lives in `test/store/database.test.ts` and `test/mcp/server.test.ts`.
- This is a follow-up to `docs/tasks/10.06.26-tool-flow-hardening/`, which already added `article_matches`.

## Scope Boundaries

- Do not add paid search, Google Programmable Search, SerpAPI, Bing Search API, or any external search billing path.
- Do not add embeddings, vector search, graph search, rerankers, or server-side model calls in this patch.
- Do not add a new MCP tool.
- Do not change corpus ingestion or production deployment behavior.
- Keep this patch backward compatible by making the new field optional.

## Assumptions

- `search_laws` remains a candidate discovery tool, not a final legal-answer tool.
- `get_article` remains the authoritative path for full article text before answering legal questions with citations.
- `article_matches[0]` is the best available local routing hint for the first follow-up call.
- The fastest reliable model guidance is a copy-ready structured `next_tool` field in `search_laws` output.

## Risks / Notes

- This does not force ChatGPT to call the tool. It improves model guidance through structured output that the model can reuse directly.
- Host-level safety blocks remain outside this MCP server's control.
- The field must stay optional to avoid breaking existing clients that only read `citation`, `snippet`, `score`, `matched_fields`, or `article_matches`.
- If future work explores graph or vector search, it should be planned separately and must remain self-hosted/local unless explicitly approved.
